## Dan Harmon's Story Circle 구조 매핑

기존 5단계 구성(문제 발견 → 기획 회의 → 바이브코딩 → 결과물 시연 → 평가/클로징)에
Story Circle 8단계 서사 아크를 오버레이합니다.

```
        ① YOU (일상)
       /          \
    ⑧ CHANGE      ② NEED
      |                |
    ⑦ RETURN      ③ GO
      |                |
    ⑥ TAKE        ④ SEARCH
       \          /
        ⑤ FIND
```

| Story Circle | 의미 | EP.01 적용 | 구성 파트 | 시간 |
|:---:|------|------|------|------|
| ① YOU | 편안한 일상 | JAY의 평범한 캠퍼스 일상, 카페에서 코딩하는 모습 | 인트로 | 0:00~0:15 |
| ② NEED | 욕구/문제 발생 | 카페에서 전 여친과 마주침! "이 비극을 끝내겠다" | 문제 제시 | 0:25~1:05 |
| ③ GO | 낯선 세계 진입 | Claude Code 터미널을 열고 바이브코딩 시작 | 기획 + 바이브코딩① | 1:05~2:30 |
| ④ SEARCH | 시련과 적응 | 에러 폭주, GPS 권한 오류, 시행착오의 연속 | 바이브코딩② | 2:30~3:30 |
| ⑤ FIND | 발견/성취 | 앱 완성! 지도에 위험 반경이 표시되는 순간 | 바이브코딩② 후반 | 3:30~4:05 |
| ⑥ TAKE | 대가/반전 | 실제 테스트 — 앱이 작동하지만 예상 못한 웃긴 결과 발생 | 결과물 시연 | 4:05~4:50 |
| ⑦ RETURN | 일상 복귀 | 쓸모 판정 + "결국 이게 제일 쓸데없었습니다" | 평가/클로징 | 4:50~5:15 |
| ⑧ CHANGE | 변화된 나 | 하지만 시청자는 "나도 만들 수 있겠다"는 깨달음 | 아웃트로 | 5:15~5:30 |

## 상세 씬별 기획

### ① YOU — 일상의 편안함 (0:00~0:15) / 인트로

**서사 목표:** 시청자에게 "이 사람은 평범한 대학생이다"라는 동일시를 만든다.

| 항목 | 상세 |
|------|------|
| **화면 구성** | 터미널 화면 풀스크린. 커서가 깜빡이다가 타이핑 시작 |
| **타이핑 텍스트** | `$ 오늘의 하찮은 문제:` → (줄바꿈) → `"헤어진 여자친구랑 같은 카페에서 마주치는 비극"` |
| **자막/그래픽** | 코드 에디터 스타일 폰트, 초록색 텍스트, 타이핑 한 글자씩 |
| **BGM** | lo-fi 비트 인트로 (페이드 인, 볼륨 30%) |
| **효과음** | 기계식 키보드 타건음 ASMR (Cherry MX Blue 느낌) |
| **얼굴캠** | PIP(우하단), JAY가 타이핑하며 씩 웃는 표정 |
| **감정 곡선** | 평온 → 약간의 호기심 유발 |

**촬영 체크리스트:**
- [ ] OBS 터미널 화면 녹화 세팅 (1080p, 60fps)
- [ ] 타이핑 애니메이션 스크립트 준비 (타이핑 속도: 보통)
- [ ] 얼굴캠 PIP 위치 확인 (우하단, 화면의 1/6 크기)
- [ ] 키보드 타건음 녹음 또는 효과음 파일 준비

### ② NEED — 비극의 발생 (0:25~1:05) / 문제 제시

**서사 목표:** "누구나 겪어본" 경험에서 공감을 폭발시킨다. Story Circle에서 가장 중요한 단계 — 시청자가 "맞아, 이거 진짜 싫어"라고 느껴야 ③ GO로의 전환이 설득력을 가진다.

| 항목 | 상세 |
|------|------|
| **씬 A (0:25~0:45)** | **재연 영상** — 편의점/카페에서 전 여친과 마주치는 상황 코믹 재현 |
| 화면 | JAY가 카페에 들어섬 → 자리에 앉으려는데 → 맞은편에 전 여친(친구 섭외) 발견 → 프리즈 프레임 |
| 연출 | 발견 순간 = 슬로모션 + 호러 줌인 (Hitchcock Zoom 패러디) |
| 효과음 | 공포 영화 "쨍-" 현악 스팅어 + 심장 박동 소리 |
| 자막 | (화면 중앙, 빨간 글씨) **"어... 안녕..."** |
| **씬 B (0:45~1:05)** | **얼굴캠 토크** — 책상 앞에서 시청자에게 직접 호소 |
| 대사 (예시) | "여러분도 이런 경험 있잖아요. 같은 동네, 같은 학교... 헤어지면 끝이어야 하는데 자꾸 마주치는 거. 편의점에서 딱 눈 마주치는 그 순간... 아 진짜 소름이에요." |
| 화면 | 얼굴캠 풀샷 + 관련 밈/짤 PIP 삽입 |
| BGM | 긴장감 있는 서스펜스 비트 (lo-fi 변형) |
| **감정 곡선** | 공감 + 약간의 고통 → "이걸 해결할 수 있다면..." 으로 전환 |

**촬영 체크리스트:**
- [ ] 재연 장소 섭외 (카페 or 편의점, 사람 적은 시간대)
- [ ] 전 여친 역할 친구 섭외 (연기 필요 없음, 앉아만 있으면 됨)
- [ ] 프리즈 프레임 + 줌인 편집 포인트 마킹
- [ ] 호러 스팅어 효과음 파일
- [ ] 토크 파트 대사 리허설 (자연스러운 말투 연습)

### ③ GO — 낯선 세계로의 진입 (1:05~2:30) / 기획 + 바이브코딩 ①

**서사 목표:** 시청자와 함께 "미지의 영역"에 발을 딛는다. 코딩을 모르는 사람이 AI에게 말로 시키는 순간 = 모험의 시작.

#### 파트 A: 기획 (1:05~1:35)

| 항목 | 상세 |
|------|------|
| **핵심 대사** | **"대딸깍의 시대, 제가 해결해보겠습니다"** (시그니처 대사) |
| **화면** | 노트 앱 / iPad에 손으로 앱 기능 스케치하는 모습 |
| 스케치 내용 | ① 전 여친 위치 감지 ② 위험 반경 설정 (500m) ③ 접근 시 경고 알림 |
| **연출** | 와이어프레임을 그리면서 설명. 손그림 느낌 → 나중에 완성된 앱과 대비 효과 |
| **효과음** | 아이디어 전구 "팅!" 소리 (스케치 각 항목 등장 시) |
| **BGM** | 경쾌한 lo-fi 비트 (볼륨 업) |
| **자막** | 핵심 기능 3가지를 순서대로 텍스트 오버레이 |
| **감정 곡선** | "어? 이거 되나?" 기대감 상승 |

#### 파트 B: 바이브코딩 ① (1:35~2:30)

| 항목 | 상세 |
|------|------|
| **화면** | 스크린캐스트 (코드 에디터/터미널) + 얼굴캠 PIP |
| **내용** | Claude Code에 자연어로 첫 지시 → 기본 구조 생성 |
| 프롬프트 예시 | "두 사람의 GPS 위치를 비교해서 500m 이내로 접근하면 경고해주는 앱 만들어줘" |
| **연출 포인트** | AI가 코드를 쫙 생성하는 순간 = 경이로움 + 웃김 ("와 이게 돼?") |
| 자막 강조 | AI 프롬프트와 응답을 자막으로 하이라이트 |
| **효과음** | 코드 생성 시 "따다닥-" 타이핑 가속음 |
| **BGM** | lo-fi 코딩 비트 (메인 테마) |
| **감정 곡선** | 경이 → 기대 → "이거 진짜 되는 거야?" |

**촬영 체크리스트:**
- [ ] Claude Code 사전 테스트 (실제 프롬프트 리허설)
- [ ] OBS 화면 녹화 레이아웃 (터미널 + 얼굴캠 PIP)
- [ ] 스케치용 노트 앱 또는 종이+펜 준비
- [ ] AI 프롬프트 대사 스크립트 준비

### ④ SEARCH — 시련과 적응 (2:30~3:30) / 바이브코딩 ②

**서사 목표:** 영웅의 여정에서 가장 재미있는 구간. 실패와 좌절이 없으면 성공도 감동이 없다. **여기서 시청자가 가장 많이 웃어야 한다.**

| 항목 | 상세 |
|------|------|
| **첫 번째 시행착오** | GPS 권한 오류 — 위치를 못 가져옴 |
| 리액션 | "아 왜 안 돼... Claude야 이거 왜 이래" (당황 리액션) |
| 화면 | 에러 메시지 클로즈업 → JAY 얼굴 리액션 컷 |
| 효과음 | "삐----" 에러 알림음 + 코믹 실패 사운드 (트롬본 "빠빠바~") |
| **두 번째 시행착오** | 앱이 작동은 하는데 반경 설정이 5km로 됨 (서울 반이 위험 구역) |
| 리액션 | "야 이러면 서울을 못 돌아다니잖아" (웃으면서 수정 요청) |
| 화면 | 지도에 거대한 빨간 원이 서울 전체를 덮는 모습 → 크게 웃음 |
| 효과음 | 경고 알림 폭주 소리 → 핵폭발 비주얼 이펙트 (밈 느낌) |
| **수정 과정** | 타임랩스로 압축 (3~4배속) |
| 자막 | "수정 중..." + 경과 시간 카운터 오버레이 |
| BGM | 긴장감 있는 비트 → 해결될 때 비트 드롭 |
| **감정 곡선** | 좌절 → 짜증+웃김 → 다시 도전 → 긴장감 |

**촬영 체크리스트:**
- [ ] 시행착오 장면 의도적으로 남겨두기 (사전에 에러를 연출하거나, 자연 발생 에러 활용)
- [ ] 리액션 오버하지 않기 (자연스럽게, 채널 톤은 무심한 듯 진지)
- [ ] 타임랩스 구간 편집점 확보
- [ ] 서울 지도 위 거대 반경 화면 캡처

### ⑤ FIND — 발견의 순간 (3:30~4:05) / 바이브코딩 ② 후반

**서사 목표:** 카타르시스의 정점. "진짜 됐다!"의 순간을 극대화한다.

| 항목 | 상세 |
|------|------|
| **결정적 순간** | 수정 완료 → 앱 빌드 성공 → 지도에 500m 위험 반경이 깔끔하게 표시됨 |
| 대사 | "어...? 어어어?! 된다! 야 이거 진짜 된다!" |
| **화면** | 앱 화면 풀스크린 → JAY 리액션 컷 (진심으로 놀라는 표정) |
| **BGM** | 비트 드롭 + 성공 팡파레 (짧게 2~3초) |
| **효과음** | "딩동댕-!" 성공 사운드 + 환호 |
| **편집** | 슬로모션 리플레이 — 빌드 성공하는 터미널 화면을 한 번 더 보여줌 |
| **자막** | **BUILD SUCCESS** (초록색, 터미널 스타일로 화면 중앙에 크게) |
| **감정 곡선** | 긴장 → **폭발적 해소** (카타르시스) |

**촬영 체크리스트:**
- [ ] 빌드 성공 순간 리액션 자연스럽게 (리허설하되 과하지 않게)
- [ ] 앱 화면 캡처 (지도 + 위험 반경 시각적으로 보기 좋게)
- [ ] 성공 팡파레 효과음 준비

---

### ⑥ TAKE — 대가와 반전 (4:05~4:50) / 결과물 시연

**서사 목표:** Story Circle의 핵심 — 원하는 것을 얻었지만 예상치 못한 대가가 따른다. **여기서 웃기면서도 "쓸데없는 앱"이라는 채널 정체성을 확정한다.**

| 항목 | 상세 |
|------|------|
| **시연 준비** | 친구에게 전 여친 역할 부탁. 서로 다른 위치에서 시작 |
| **테스트 ①** | 500m 밖 — "현재 안전합니다" (초록 표시) → JAY 안도 |
| **테스트 ②** | 친구가 접근 → 500m 진입 → **"⚠ 주의! 전 여친 500m 이내 접근 중"** 알림 |
| 리액션 | "오 진짜 울린다!" (놀람 + 기쁨) |
| **테스트 ③ (반전)** | 100m 접근 → **"🚨 대피 경보! 즉시 이동하세요"** → JAY가 실제로 뛰어서 도망 |
| 리액션 | 뛰면서 "아 이거 쓸데없는 건 맞는데 심장이 진짜 뛰잖아...!" |
| **최종 반전** | 도망치다가 결국 다른 방향에서 친구와 마주침 → "...안녕" (어색한 웃음) |
| 자막 | **"앱은 완벽했다. 하지만 골목길은 완벽하지 않았다."** |
| **화면 구성** | 앱 화면 녹화(지도+경고) + 실외 촬영(JAY 이동) + 얼굴캠 리액션 |
| **BGM** | 기대감 → 경고 단계별 긴장 고조 → 반전 시 코믹 BGM |
| **효과음** | 500m: 부드러운 알림 / 100m: 사이렌 / 마주침: "짠-" 효과음 |
| **감정 곡선** | 기대 → 긴장 → 성공의 쾌감 → **반전(웃음)** → "역시 쓸데없다" |

**촬영 체크리스트:**
- [ ] 시연 장소 사전 답사 (GPS 수신 잘 되는 야외)
- [ ] 친구 동선 사전 기획 (500m → 100m 접근 경로)
- [ ] 스마트폰 2대 준비 (JAY + 친구 각각 앱 설치)
- [ ] 실외 촬영 시 오디오 (무선 마이크 또는 후시 녹음)
- [ ] 도망치는 장면 촬영 (핸드헬드/짐벌)
- [ ] 반전 마주침 장면 구도 기획

### ⑦ RETURN — 일상 복귀 (4:50~5:15) / 평가 & 클로징

**서사 목표:** 모험이 끝나고 다시 책상 앞으로 돌아온다. 하지만 시작할 때와는 다른 사람이다.

| 항목 | 상세 |
|------|------|
| **화면** | 다시 데스크 셋업 얼굴캠 (처음과 같은 프레이밍 = 원형 구조 완성) |
| **쓸모 판정** | 5점 만점 중 **?점** (터미널 스타일 점수판 그래픽) |
| 대사 (예시) | "쓸모 판정 들어갑니다. 이번 앱은... 기술적으로는 작동했습니다. 근데 결국 골목길에서 마주쳤으니까... **5점 만점에 1.5점**. 이별 앞에서 기술은 무력합니다." |
| **점수 공개 연출** | 드럼롤 → 점수 등장 (터미널 스타일 ASCII 아트) → 효과음 |
| **다음 예고** | "다음 하찮은 문제는..." → 힌트 텍스트만 살짝 (호기심 유발) |
| **BGM** | 클로징 BGM (메인 테마 변형, 따뜻한 톤) |
| **감정 곡선** | 만족감 + 웃음의 여운 → 다음 회 기대감 |

**촬영 체크리스트:**
- [ ] 점수판 그래픽/애니메이션 사전 제작
- [ ] 드럼롤 효과음
- [ ] 다음 에피소드 힌트 텍스트 확정

## 핵심 정리: Story Circle이 이 영상에 주는 효과

1. **공감 → 모험 → 좌절 → 카타르시스 → 반전 → 여운** 구조로 5분 30초 안에 완전한 서사 아크 완성
2. ④ SEARCH (시행착오)가 없으면 코딩 영상은 "그냥 잘 만든다"로 끝남 → **의도적 실패가 재미의 원천**
3. ⑥ TAKE (반전)이 없으면 "쓸데없는 앱"이 아니라 "유용한 앱"이 됨 → **반전이 채널 정체성의 핵심**
4. ①과 ⑧의 동일 프레이밍 → 시청자에게 **"원래 자리로 돌아왔지만 뭔가 달라진" 감각** 전달
