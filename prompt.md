# Claude Code 프롬프트: 1인 미디어 유튜브 채널 기획안 HTML 제작

## 요청 개요
세종대학교 미디어커뮤니케이션학과 <영상제작워크샵(ft. 1인미디어종합설계)> 2026년 1학기 과제용 **유튜브 채널 기획안**을 **단일 HTML 파일**로 제작해주세요. 이 문서는 교수님께 제출하는 공식 기획안이면서 동시에 채널 운영의 실질적 로드맵 역할을 합니다.

## 디자인 방향

### 전체 톤
- **에디토리얼 매거진 스타일** + 약간의 **플레이풀한 요소** (채널 컨셉인 '하찮은 앱'과 어울리는 위트)
- 다크 테마 베이스 (개발자 터미널 느낌) + 포인트 컬러로 네온 그린/옐로우 (바이브코딩의 에너지감)
- 전체적으로 깔끔하되 곳곳에 코드 에디터, 터미널 UI 요소를 모티프로 활용

### 타이포그래피
- 한글 제목: **Pretendard** 또는 **Noto Sans KR** (700, 900 웨이트)
- 영문/코드: **JetBrains Mono** 또는 **Fira Code** (모노스페이스, 개발 감성)
- Google Fonts CDN으로 임포트

### 색상 팔레트 (CSS Variables)
```
--bg-primary: #0d1117 (GitHub Dark 계열)
--bg-secondary: #161b22
--bg-card: #1c2128
--text-primary: #e6edf3
--text-secondary: #8b949e
--accent-green: #39d353 (네온 그린)
--accent-yellow: #f0c000
--accent-pink: #f778ba
--border: #30363d
```

### 레이아웃
- 최대 너비 1200px, 중앙 정렬
- 섹션 간 충분한 여백 (80px+)
- 카드 기반 레이아웃, 적절한 border-radius와 미세한 glow 효과
- 스크롤 시 섹션 fade-in 애니메이션 (IntersectionObserver 사용)
- 인쇄 시 깔끔하게 나오도록 @media print 스타일 포함

### 섹션 2: 기획의도 (컨셉 상세)

카드 레이아웃으로 다음 내용을 서술체로 작성:

**시대적 배경:**
2024년 SORA 등장 이후 AI 크리에이터 시대가 열렸다. 코딩 영역에서도 '바이브코딩(Vibe Coding)'이라는 새로운 패러다임이 등장해, 자연어로 AI에게 지시하면 앱이 만들어지는 시대가 되었다. 그러나 대부분의 개발 채널은 여전히 진지하고 기술 중심적이며, 일반인이 접근하기 어려운 분위기다.

**문화원형 분석:**
호소다 다카히로(2023)의 정의에 따르면 컨셉은 "전체를 관통하는 새로운 관점"이다. 본 채널의 관점은 **"기술의 민주화를 유머로 증명한다"**는 것이다. 임종수(2006, 2011)의 문화원형 이론에 근거하면, 이 채널이 자극하는 인간 본연의 욕망은 '창조의 즐거움'과 '웃음을 통한 공감'이다. Jung의 12가지 archetype 중 Jester(유머와 현재를 즐기려는 욕구)와 Creator(새로운 것을 만들려는 욕구)의 교차점에 위치한다.

**컨셉 방법론 적용 (배드 BAD, 2020):**
- 대의명분(정당성): "코딩 못해도 아이디어만 있으면 된다"는 AI 시대의 메시지
- 구원자(욕망): 시청자의 "나도 만들어볼까?" 욕망 자극
- 의외성(매력): '진지한 개발' 클리셰를 깨는 반전 — 쓸모없는 앱을 진지하게 만드는 아이러니
- 스피커(공감): 일상의 미세한 불편함에 대한 보편적 공감

**경쟁 환경 분석:**
피식대학이 지상파 개그 프로그램 폐지(2020, 개콘) 후 빈 자리를 채운 것처럼(BM① 경쟁이 사라진 곳에 세운 새로운 성), '바이브코딩 × 예능'이라는 니치는 한국 유튜브에서 사실상 블루오션이다. 개발 채널(코딩애플, 노마드코더 등)은 진지하고, 예능 채널은 코딩을 다루지 않는다. 이 사이의 빈 공간을 차지한다.

### 섹션 3: 포맷 상세

**포맷의 3요소** (수업 자료 기반 — 양성희, 2023):

**1) 변하지 않는 그릇 (기본 요소):**
- 인트로: 터미널 화면에 `$ 오늘의 하찮은 문제:` 타이핑 애니메이션 → 문제 텍스트 출력
- 시그니처 대사: "이걸 앱으로 만들면 되는 거 아닌가?" (매 에피소드 반복)
- 엔딩: 완성된 앱 QR코드 + "다음 하찮은 문제에서 만나요" 클로징
- BGM: lo-fi 코딩 비트 (저작권 프리)
- 자막 스타일: 코드 에디터 폰트 + 초록색 하이라이트

**2) 구조화된 서사 형식:**
매 에피소드 5단계 구조:
① [문제 발견] 일상에서 하찮은 불편함 포착 (1~2분, 공감 유발)
② [기획 회의] "이걸 앱으로 만들면?" 아이디어 스케치 (1분)
③ [바이브코딩] AI에게 말로 지시하며 앱 제작 (3~5분, 핵심 파트, 시행착오 + 웃긴 순간)
④ [결과물 시연] 완성된 앱 테스트, 실제 사용 (2분)
⑤ [평가 & 클로징] 쓸모 판정 + 다음 예고 (1분)

**3) 소재 소화력:**
"일상의 하찮은 문제"라는 프레임은 소재가 무한. 음식, 연애, 학교, 날씨, 건강, 쇼핑, 교통 등 모든 카테고리에서 소재 추출 가능. 포맷이 다양한 소재를 소화할 수 있는 유연한 그릇.

### 섹션 4: 로그라인 분석 (6요소 테이블)

### 섹션 5: 콘텐츠 로드맵 (파일럿 + 시즌 계획)

타임라인 또는 카드 그리드로 표현:

**파일럿 (1~2주차, 3~5개 동시 업로드 후 반응 탐색):**
1. "여자친구 카톡 답장 안 하면 알림 폭탄 보내는 앱"
2. "교수님 PPT 남은 슬라이드 수 예측 앱"  
3. "오늘 점심 뭐 먹을지 정해주는 극한의 결정장애 앱"
4. "친구들 약속 장소 중간지점 자동 계산 앱"
5. "수업 중 졸면 진동으로 깨워주는 앱"

**시즌1 (3~7주차):** 파일럿 반응 기반 방향 확정, 주 1회 롱폼 + 쇼츠
**시즌2 (8~12주차):** 시청자 제보/요청 기반 "하찮은 의뢰" 코너 추가
**파이널 (13~15주차):** 한 학기 베스트 앱 모음 + 실제 사용 후기 + 구독자 참여 콘텐츠

**쇼츠 전략 (거시/미시 콘텐츠 구조, 임종수 2006):**
- 롱폼 = 거시 콘텐츠 (전체 제작 과정)
- 쇼츠 = 미시 콘텐츠 (하이라이트, 웃긴 순간, 완성 결과물 30초 시연)
- 하나의 롱폼에서 2~3개의 쇼츠 추출

### 섹션 6: 레퍼런스 채널/콘텐츠 분석 (3개 이상)

카드 레이아웃으로 각 레퍼런스의 유사점/차별점 분석:

**1. 침착맨 — "병맛 실험실"**
- 유사점: Jester×Creator 문화원형, 포맷의 유연한 소재 소화력, 캐릭터 중심
- 차별점: 본 채널은 매 에피소드 실제 결과물(앱)이 나옴. 침착맨은 토크/게임 중심.
- 시사점: 캐릭터 브랜딩의 중요성, 피식대학처럼 유니버스 확장 가능성

**2. 긱블 (Geekble) — "엔지니어링 예능"**
- 유사점: 기술을 예능으로 풀어내는 포맷, "만들기"가 핵심
- 차별점: 긱블은 하드웨어/물리적 제작, 본 채널은 소프트웨어/AI 코딩. 긱블은 팀 기반, 본 채널은 1인.
- 시사점: 기술 콘텐츠도 예능화하면 넓은 타겟에 어필 가능

**3. 피식대학 — "경쟁이 사라진 곳에 세운 새로운 성"**
- 유사점: BM① 블루오션 전략, 유니버스 구축, 다양한 코너/포맷 실험
- 차별점: 피식대학은 코미디 상황극, 본 채널은 실제 제작 과정이 콘텐츠
- 시사점: 파일럿 → 반응 탐색 → 시리즈화 전략 벤치마킹

**4. 노마드코더 / 코딩애플 — "개발 교육 채널"**
- 유사점: 코딩/개발 소재
- 차별점: 이들은 교육 목적의 진지한 톤, 본 채널은 예능 목적의 유쾌한 톤. 바이브코딩 특화.
- 시사점: 기존 개발 채널과의 명확한 차별화 포인트

### 섹션 7: 수익모델 & 마케팅

**수익모델 (수업 자료 기반 7가지 중 적용 가능한 것):**
1. 광고 수익(애드센스): YPP 달성 목표 (구독자 500명 + 시청시간 3,000시간 또는 쇼츠 300만 조회)
2. 제휴 마케팅: AI 도구(Claude, Cursor 등) 레퍼럴 링크
3. 스폰서 콘텐츠: AI/개발 도구 기업과의 브랜디드 콘텐츠 가능성
4. 컨설팅/강좌: 바이브코딩 노하우 공유 (중장기)

**마케팅 전략:**
- 쇼츠 우선 전략: 일일 2,000억 조회 쇼츠 시장 공략, 30~60초 임팩트 클립으로 유입
- 커뮤니티 활용: 시청자 "하찮은 문제 제보" → 콘텐츠화 → 참여 유도
- 크로스 플랫폼: 틱톡/인스타 릴스에 쇼츠 동시 업로드
- SEO: "바이브코딩", "AI 앱 만들기", "쓸모없는 앱" 등 키워드 선점

**알고리즘 전략 (수업 자료 기반):**
- 8분룰 적용: 롱폼 8분+ 제작하여 미드롤 광고 게재 가능
- 시청지속률 목표: 8~12분 영상 기준 40% 이상 (약 3~5분 시청)
- CTR 최적화: 썸네일에 "완성된 앱 스크린샷 + 리액션 얼굴 + 자극적 텍스트" 공식
- 초기 전략: 짧은 영상부터 시작 → 충성 구독자 확보 후 점차 길이 확장

## 기술 요구사항

1. **단일 HTML 파일** (CSS, JS 모두 인라인)
2. **Google Fonts** 사용 (Pretendard or Noto Sans KR + JetBrains Mono)
3. **반응형** (모바일/태블릿/데스크톱)
4. **스크롤 애니메이션**: IntersectionObserver로 섹션 fade-in
5. **인쇄 최적화**: @media print로 배경색 제거, 폰트 색 검정, 레이아웃 단순화
6. **네비게이션**: 좌측 사이드바 또는 상단 sticky nav로 섹션 이동
7. **인터랙티브 요소**: 
   - 로그라인 6요소를 호버하면 상세 설명 표시
   - 콘텐츠 로드맵을 타임라인 형태로 시각화
   - 구성안 테이블에서 각 구간 클릭 시 상세 설명 expand
8. **코드 에디터 모티프**: 일부 섹션을 코드 블록 스타일로 표현 (라인넘버, 신택스 하이라이팅 느낌)
9. 총 길이: 스크롤 시 약 8~10 화면 분량의 풍성한 문서
